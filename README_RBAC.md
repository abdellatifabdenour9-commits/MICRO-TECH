## 💻 أمثلة الاستخدام

### مثال 1: التحقق من الدور

```typescript
import { useAuth } from './context/AuthContext';

function AdminButton() {
  const { isAdmin } = useAuth();
  
  if (!isAdmin()) {
    return null; // لا تعرض الزر للعملاء
  }
  
  return <button>اذهب إلى لوحة التحكم</button>;
}
```

### مثال 2: التحقق من الصلاحية

```typescript
import { usePermission } from './context/AuthContext';
import { PERMISSIONS } from './authRoles';

function ProductManager() {
  const hasAccess = usePermission(PERMISSIONS.MANAGE_PRODUCTS);
  
  if (!hasAccess) {
    return <div>ليس لديك صلاحية</div>;
  }
  
  return <ProductForm />;
}
```

### مثال 3: حماية المسار

```typescript
import { ProtectedRoute } from './components/ProtectedRoute';

function AdminPage() {
  return (
    <ProtectedRoute requiredRole="admin">
      <AdminDashboard />
    </ProtectedRoute>
  );
}
```

## 🔑 الأدوار المتاحة

### 1. 👑 المسؤول (Admin)
- الصلاحيات: كاملة
- الوصول: جميع الصفحات
- مثال البريد: `admin@micro-tech.com`

### 2. 👔 الموظف (Employee)
- الصلاحيات: معالجة الطلبات والدعم
- الوصول: الطلبات والدعم والمتجر
- مثال البريد: أي `@micro-tech.com`

### 3. 👨‍🏫 المعلم (Teacher)
- الصلاحيات: إدارة الدورات
- الوصول: المتجر والدورات
- مثال البريد: أي بريد مع اختيار Teacher

### 4. 👤 العميل (Customer)
- الصلاحيات: الشراء والتسوق
- الوصول: المتجر والسلة والطلبات
- مثال البريد: أي بريد عادي

### 5. 👁️ الزائر (Guest)
- الصلاحيات: قراءة فقط
- الوصول: الصفحات العامة فقط
- الوصول: بدون تسجيل دخول

## 🐛 استكشاف الأخطاء

### المشكلة: لا أستطيع الدخول كمسؤول

**الحل:**
```
1. تأكد من كتابة البريد بشكل صحيح: admin@micro-tech.com
2. تأكد من كتابة كلمة المرور: admin123
3. افتح وحدة التطوير F12 وتحقق من الأخطاء
4. افرغ localStorage واحاول مرة أخرى
```

### المشكلة: لوحة التحكم لا تظهر

**الحل:**
```
1. تأكد من تسجيل الدخول كمسؤول
2. تحقق من الدور: user.role === 'admin'
3. افحص localStorage للتأكد من حفظ البيانات
4. تحقق من console من أي أخطاء
```

### المشكلة: الصلاحيات لا تعمل

**الحل:**
```
1. استورد PERMISSIONS بشكل صحيح
2. استخدم usePermission Hook
3. تحقق من أن الدور له هذه الصلاحية
4. افحص authRoles.ts للصلاحيات
```
